    cookie和session

    1.cookie

  2. 通过cookie可以向客户端存储数据
  3. url重写可以解决用户禁用cookie的情况下依然可以保持客户端和服务端的通信状态(这个是后台写的)
  4. document.cookie可以访问cookie;
  5. cookie的格式:键值对;
  6. 多个cookie之间使用分号和空格隔开;例如:username=zhangsan; age=12;
  7. document.cookie = 'username=zhangsan'在浏览器中是有缓存的,关闭浏览区可以清除;

    2.cookie的分类

  8. session cookie 会话cookie:关闭浏览器之后自动清除,但是可以增加其有效期,以下例子增加了5天的有效期

  9. persistant cookie 持久cookie:关闭浏览器

  10. 获得cookie,封装函数
  11. 在子路径设置的cookie,在父路径下是获取不到的;反过来可以
    不可以跨域获取cookie,但是可以跨页面共享;
  12. 可以用jquery-cookie插件应用;
  • encodeURLComponent该方法会对url地址中的特殊字符进行编码(当参数中包含URL特殊字符的时候可以使用该方法进行编码处理);
  • //获取sessionID
    console.log($.cookie(‘PHPSESSID’));

    3. $.cookie()使用

  1. 新添加一个cookie会话;$.cookie('username','zhangsan');
  2. 创建cookie并设置有效期为7天;$.cookie('age','18',{expires:7});此时的cookie称作持久cookie,即persistant cookie;
  3. 创建cookie并设置cookie的有效路径$.cookie('username','lisi',{expires:7,path:'/'});
    • 在默认情况下,只有cookie的页面才能读取该cookie信息,如果想让另一个页面读取该cookie信息,必须设置cookie的路径;cookie的路径用于设置能够读取cookie的顶级目录,即根目录,可以让当前项目下所有的网页都能相互读取cookie,
  4. raw:true;在读写cookie时自动使用encodeURLComponen进行编码和解码;
  5. 删除cookie可以使用null作为cookie的值就可以$.cookie('username',null)
