Git 基础知识

发表于 2016-06-13 | 分类于 Diary

git day02 git使用

本地仓库

三个区域

  • 工作目录
  • 暂存区
  • 本地仓库

    文件的四个状态

  • 未跟踪 untracked
  • 已暂存 staged
  • 已提交 coomited
  • 已修改 modified

    相关命令

  • git add file
    • 将未跟踪的文件加到暂存区
    • 把暂存区修改的内容从新提交到暂存区
  • git commit -m "备注"
    • 将暂存区的内容提交到本地仓库中
  • git rm --cached files
    • 将暂存的文件移动到工作目录
  • git checkout files
    • 暂存区中修改的的内容撤销掉
  • git reset HEAD(表示本地仓库中最后一次提交,commitId)
    • --hard工作目录、暂存区、本地仓库都会变化;
    • --soft只会回滚本地仓库,工作目录和暂存区不会变化
    • --mixed默认的,用本地仓库的版本覆盖暂存区文件,也会回滚本地仓库,不会修改工作目录

      远程仓库(共享仓库)

  • git clone克隆远程仓库全部内容
  • git pull从远程仓库中下载最新版本,覆盖本地分支内容
    • git pull 远程仓库地址 远程分支:本地分支
  • git push提交本地代码版本
    • git push 远程仓库地址 本地分支:远程分支
  • git fetch从远程仓库下载最新版本内容,但是不合并,需要手动使用`git merge
    合并;
  • git remote查看远程分支别名
  • gitremote add origin添加远程仓库地址的别名
  • git remote show origin查看远程仓库别名的二聚体地址

    分支操作

  • git branch 分支名称 创建分支
  • git branch -r查看远程分支
  • git branch -a查看所有分支
  • git checkout 分支名称 切换分支
  • git checkout -b 分支名称 创建并切换分支
  • git branch -d 分支名称 删除本地分支
  • git push origin --delete 删除远程分支
  • git merge 来源分支 合并分支
  • git stash 保存当前版本状态
  • git stash pop恢复保存的版本状态
